# Justificación Metodológica

Este documento presenta una breve justificación metodológica utilizada en el desarrollo del proyecto.

## Enfoque Metodológico

Se optó por una metodología ágil, basada en iteraciones cortas y entregas incrementales. Esto permitió adaptar el desarrollo a los requerimientos y cambios surgidos durante el proceso, facilitando la retroalimentación continua y la mejora progresiva del producto.

## Fases del Desarrollo

1. **Análisis y planificación:** Se definieron los objetivos, funcionalidades principales y la arquitectura base del proyecto.
2. **Diseño:** Se elaboraron prototipos de la interfaz y se seleccionaron tecnologías adecuadas (HTML, CSS, JavaScript, SASS, Bootstrap).
3. **Implementación:** El desarrollo se realizó en módulos, priorizando la separación de responsabilidades y la reutilización de componentes.
4. **Pruebas:** Se realizaron pruebas funcionales y de usabilidad en diferentes dispositivos para asegurar la calidad y accesibilidad.
5. **Documentación y entrega:** Se generó documentación técnica y de usuario, así como evidencias de funcionamiento.

## Decisiones Técnicas

- **Uso de SASS y Bootstrap:** Para agilizar el desarrollo de estilos y asegurar una interfaz responsiva.
- **Estructura modular en JavaScript:** Para facilitar el mantenimiento y escalabilidad del código.
- **Persistencia en JSON:** Para simular almacenamiento de datos y facilitar pruebas.

## Herramientas y Recursos

- Visual Studio Code
- Git y GitHub para control de versiones
- Figma para diseño de interfaces
- Navegadores web para pruebas cruzadas

## Conclusión

La metodología seleccionada permitió cumplir con los objetivos del proyecto, adaptarse a cambios y entregar un producto funcional, documentado y probado en distintos entornos.
